Mini Portfolio: Yuri Vishnesky

Hello!

I don't have a portfolio on my website at the moment, so I put together this quick tour through four of my recent projects.

1. Silk

Silk is a painting program that enables anyone in the world to make beautiful art.

Favorite quote: "This is far too soothing. If Genghis Khan had had access to that, the Mongols would have stayed home."

The new Silk 2 app for iOS is the best take on Silk's ideas so far.

Specific interesting contributions

  • Responsible for all design and implementation. Silk is my perennial toy project – it's a design sandbox where I get to explore the limits of user interface design and dynamic interaction.

  • Over 25,000,000 people have played with the website. Artists use it for creative inspiration, teachers use it to teach symmetry.

  • On exhibit in the Children’s Creativity Museum in San Francisco for several years.

  • Featured twice by Apple on the front page of the App Store, and selected by Apple for a national cross-promotion in Sephora stores.

Tools

  • The (several-years-old) website is Coffeescript + Canvas. The color picker (blue circle; top left) uses d3 for the color blending interface — try dragging from one color to another.
  • The new Silk 2 app is Swift 3 + Metal.

Links

  • Google News Lab post
  • Writeup in Scientific American
  • Moritz Stefaner's writeup about the project

2. Business Cards

I decided to make myself some beautiful business cards. The project quickly blossomed to ridiculous proportions and I ended up making 600 individually unique letterpressed fine art prints. Totally worth it. :-)

Specific interesting contributions

  • Designed a walkthrough article for the project, including live-replays of the generative algorithm so that you can see how the final structure emerges from the individually placed pixels.

  • Implemented a cool pixel-placing algorithm in Julia.

  • Published an open-source package that implements a a nearest-neighbor search algorithm by generalizing binary search.

  • Published an open-source package that implements a treap, a funny-named data structure that is both a tree and a heap.

  • Shared my experiences making art with a programming language designed for technical computing at the first-ever Julia conference in Chicago.

Tools

  • Julia for the core algorithm and asset production pipeline.
  • D3 for the art generation playback engine in the story post.
  • An early alpha version of Figma for exploring card designs and designing the story post.

3. Luminoso

I was an early employee at Luminoso, a text analytics startup spun out of the MIT Media Lab where I was responsible for designing and developing Luminoso's main product, the Luminoso Dashboard.

The Dashboard is a web-based environment that enables users to analyze and explore their data to find stories that solve real-world problems.

For example, we helped many companies make decisions about product development based on large amounts of customer feedback.

Specific interesting contributions

  • Developed novel visualizations to help decision-making by using Luminoso's vector-based text understanding technology. Became a better designer through years of focused experimentation and user feedback.

  • One specific visualization that I'm proud of is the Concept Cloud, a new take on word clouds. It's a seriously cool visualization that makes word clouds mean something — position, size, and color all matter and give you affordances to further explore the relationships between concepts.

  • Collaborated with internal analysts to help find insights in client data and to make user-driven changes to the dashboard.

  • I'm not generally a fan of software patents, but it seems worth mentioning that I'm on one for the Concept Cloud: Word cloud rotatable through N dimensions via user interface.

  • As a fun side project, I made a visualization of the xkcd color survey using Luminoso technology to find the most positive, negative, and food-related colors.

Tools

  • React for general page structure and D3 for core visualizations.
  • Webpack and Jasmine for builds and testing.

4. Google Rhythm of Food

Working together with Moritz Stefaner, I helped create a website exploring seasonal food patterns in Google searches. I was responsible for a large part of the development, as well as some parts of the design.

I joined the project in the second half of the prototyping phase, helped define the visualization and scope, and was involved in all aspects of the design and development of the final product.

Specific interesting contributions

  • Built a hybrid Canvas/SVG renderer to handle the increasing performance requirements of rendering an interactive responsive site showing 135,025 individual data points.

  • Built a simple annotation helper to assist the person responsible for text and annotation in annotating specific points on the various charts by pointing and clicking.

  • Researched and implemented a peak detection algorithm to identify foods with strong seasonal peaks around holidays. This was used to label the relevant holidays for each food — for example, here's Apple Cider.

  • Built an animation pipeline to export animation frames that can be made into a .gif for sharing food animations on social media. The same animations can be run live on the site — for example, here's the page for Asparagus. Click "Play!" in the lower-right-hand corner.

  • Came up with the "Charts that look like the food" category in the explore section. :-)

Tools

  • Mathematica for data science and peak detection prototyping.
  • React components to build the site, D3 for the core of the visualizations, and mobx to bind them.
